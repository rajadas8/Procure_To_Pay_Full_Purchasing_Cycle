# Procure_To_Pay_Full_Purchasing_Cycle
🧾 Procure-to-Pay (P2P) Cycle Simulation – SAP MM (Python)

📌 Overview: This project simulates the complete Procure-to-Pay (P2P) cycle used in SAP MM. It demonstrates how organizations manage procurement from raising a request to completing vendor payment using Python.

🎯 Objective: The main goal is to understand real-world business workflows and implement them using programming. This project helps bridge the gap between technical skills and enterprise processes.

🔄 Workflow: Purchase Requisition → Approval → Vendor Selection → Purchase Order → Goods Receipt → Invoice Verification → Payment

🛠️ Tech Stack: Python is used as the core language along with pandas for data handling, datetime for timestamps, and uuid for unique ID generation.

⚙️ Features:
End-to-end procurement cycle simulation
Real SAP MM workflow understanding
Automated PR, PO, GR, Invoice, and Payment process
Inventory (stock) update after goods receipt
Invoice calculation based on quantity and price
Clean and modular Python functions

▶️ How to Run:
Clone the repository, open the project folder, and run the notebook in Jupyter step by step.
